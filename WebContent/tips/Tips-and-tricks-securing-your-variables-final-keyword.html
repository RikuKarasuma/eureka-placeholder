Tips and Tricks: Securing your variables. The Final keyword in Java.


Previously, we looked at "Securing your classes". Which highlights when and how to use to the final keyword on classes and methods in order to secure them. Today we are using the same concept to secure variables, again from a design standpoint.

The final keyword uses immutability to prevent designs from changing, either maliciously or by another developer that comes after you. There are several more uses for the keyword stated below.

Please be aware that if you did not create a security manager at the beginning of your program, then final does not matter when it comes to reflection use!

  • A final class can't be sub-classed.
  • A final method can't be overridden.
  • A final variable can only be instantiated once.
  • When an inner class is defined within a method, you must make all local method variables it uses from outside that inner class as final.

We are focusing only on its third effect today. A final variable can only be instantiated once.

Creating a variable once and labelling it as final ensures that this variable won't be changed for the lifetime of the program. If we design variables that depend on only being instantiated once, then the final keyword is extremely useful in maintaining our design choices.
